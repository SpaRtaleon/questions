Questions & Answers

1.What is a Method?

A method is a block of code that performs a specific task. You define a method once, and you can use (or "call") it whenever you want. It helps you organize and reuse code. In HTML, you don’t use methods directly, but in programming languages like JavaScript, Java, or Python, methods are essential. Example in JavaScript: function greet() { console.log("Hello, World!"); } Here, greet is a method that prints "Hello, World!" to the console.

2.Method Arguments

Arguments are the actual values you pass to a method when calling it. Example: function greet(name) { console.log("Hello, " + name); } greet("John"); // "John" is an argument Here, "John" is the argument passed to the greet method.

3. Method Parameters

Parameters are placeholders for the values (arguments) a method receives. They act like variables inside the method. Example: function greet(name) { // 'name' is a parameter console.log("Hello, " + name); } Parameter → Variable in the method definition (name). Argument → Actual value passed to the method ("John").

4.Method Call

A method call is how you execute a method. You use the method’s name followed by parentheses, optionally passing arguments. Example: greet("Alice"); // This is a method call

5. Method Return Types

The return type specifies what type of value a method gives back after execution. In JavaScript, you use the return statement to send a result. Example: function add(a, b) { return a + b; // Returning the sum of a and b } let sum = add(5, 3); console.log(sum); // Output: 8 Here, the method add returns a number. In languages like Java or Python, you explicitly declare the return type (e.g., int, string).

What is OOPS?

OOPs stands for Object-Oriented Programming. It’s a programming paradigm based on the concept of "objects" that represent real-world entities. These objects contain data (attributes or properties) and methods (functions or behaviors) that operate on the data.

Let’s break it down! The key principles of OOP are:

  • Class: A blueprint for creating objects. It defines attributes and methods.

    Example: class Car { }

  • Object: An instance of a class with its own state and behavior.

    Example: myCar = new Car();

  • Encapsulation: Bundling data and methods within a class, and restricting direct access to some components.

    Example: Using private variables with public getter/setter methods.

  • Inheritance: Allowing one class to inherit properties and methods from another class, promoting code reusability.

    Example: class ElectricCar extends Car { }

  • Polymorphism: The ability for a method to behave differently based on the object that calls it.

    Example: Method overloading and overriding.

  • Abstraction: Hiding complex implementation details and showing only the necessary features.

    Example: Abstract classes and interfaces.
